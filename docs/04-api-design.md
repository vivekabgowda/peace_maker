# 04 ŌĆö API Design

## 1. Conventions

- **Base path:** `/api/v1` (URL-versioned; breaking changes bump the version).
- **Format:** JSON; `snake_case` bodies mirroring domain models.
- **Auth:** `Authorization: Bearer <access_token>` (JWT). Refresh via cookie or
  body per client. See [12-security-compliance.md](12-security-compliance.md).
- **Errors:** consistent envelope (below).
- **Pagination:** cursor-based for feeds, `?limit=&cursor=`; page-based allowed
  for admin tables.
- **Idempotency:** mutating endpoints accept `Idempotency-Key` header.
- **Time:** all timestamps ISO-8601 UTC; client renders IST.
- **Spec:** OpenAPI 3.1 auto-generated by FastAPI; the TS client is generated
  from it, so contracts never drift.

### Standard error envelope

```json
{
  "error": {
    "code": "risk_limit_exceeded",
    "message": "Position size would exceed max risk per trade.",
    "details": { "limit_pct": 1.0, "attempted_pct": 1.8 },
    "correlation_id": "b1f2ŌĆ”"
  }
}
```

### Standard list envelope

```json
{
  "data": [ /* items */ ],
  "page": { "next_cursor": "eyJŌĆ”", "has_more": true }
}
```

## 12. Realtime (WebSocket)

Single multiplexed connection at `wss://ŌĆ”/api/v1/ws`, authenticated via a
short-lived ticket obtained from `POST /auth/ws-ticket`.

**Client ŌåÆ server** (subscribe):
```json
{ "action":"subscribe", "channels":["quotes:NIFTY","recommendations","alerts"] }
```

**Server ŌåÆ client** (event):
```json
{ "channel":"recommendations",
  "event":"new",
  "data": { "id":"ŌĆ”","instrument":"RELIANCE","confidence":78,"rank":2 },
  "ts":"2026-07-20T04:15:22Z" }
```

| Channel | Payload |
|---------|---------|
| `quotes:{symbol}` | Live LTP, change, volume |
| `indices` | Index + VIX ticks |
| `recommendations` | New / invalidated recommendations |
| `alerts` | Criteria-met notifications |
| `portfolio` | Live P&L updates |

WS is fed from Redis pub/sub; the gateway performs per-user authorization on
subscribe and heartbeats every 30 s.

## 13. Rate Limits & Versioning

| Aspect | Policy |
|--------|--------|
| Rate limit | Redis token bucket; defaults 60 req/min/user, tighter on heavy endpoints |
| Deprecation | `Sunset` + `Deprecation` headers; min 90-day window |
| Backward compat | Additive changes only within a version |
