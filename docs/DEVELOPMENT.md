# Development Guide

Everything you need to run BKN AI Capital locally and contribute to it. For the
system design, see the [Software Design Document](SOFTWARE_DESIGN_DOCUMENT.md).

## 1. Prerequisites

| Tool | Version |
|------|---------|
| Docker + Docker Compose | latest |
| Python (for local backend work) | 3.12 |
| Node.js (for local frontend work) | 20 |
| Make | any |

## 2. Quick start (Docker — recommended)

```bash
cp infra/env/.env.dev.example .env     # or: make up creates it for you
make up                                # build & start the full stack
```

Services:

| Service | URL |
|---------|-----|
| Frontend | http://localhost:3000 |
| Backend API docs | http://localhost:8000/docs |
| Backend health | http://localhost:8000/api/v1/health/ready |
| n8n | http://localhost:5678 |
| Postgres | localhost:5432 (user/pass/db = `bkn`) |
| Redis | localhost:6379 |

The backend applies migrations automatically on start. Seed an admin user:

```bash
make seed        # creates admin@bkn.local / ChangeMe!123 (override via env)
```

Useful commands: `make help`, `make logs`, `make migrate`, `make down`.

## 3. Local backend (without Docker)

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"

# Point at a running Postgres/Redis (e.g. the compose ones):
export BKN_DATABASE_URL="postgresql+asyncpg://bkn:bkn@localhost:5432/bkn"
export BKN_REDIS_URL="redis://localhost:6379/0"
export BKN_JWT_SECRET_KEY="dev-secret-change-me-1234567890"

alembic upgrade head
python -m scripts.seed
uvicorn app.main:app --reload
```

Quality gates:

```bash
ruff check app tests scripts     # lint
black --check app tests scripts  # format
mypy app                         # strict typing
pytest                           # tests + coverage (uses an isolated sqlite DB)
```

## 4. Local frontend (without Docker)

```bash
cd frontend
npm ci
npm run dev        # http://localhost:3000 (proxies /api to the backend)
```

Quality gates:

```bash
npm run lint       # eslint
npm run typecheck  # tsc --noEmit
npm run test       # vitest
npm run build      # production build
npm run format     # prettier --write
```

## 5. Database migrations (Alembic)

Migrations live in `backend/migrations/versions`. The Alembic environment is
async and uses the app settings + ORM metadata.

```bash
cd backend
alembic upgrade head                          # apply all
alembic downgrade -1                          # roll back one
alembic revision --autogenerate -m "message"  # create a migration from model changes
alembic upgrade head --sql                    # render SQL without applying (review)
```

Guidelines:
- Migrations are **forward-only** in shared environments; review autogenerated
  output before committing (autogenerate is a starting point, not gospel).
- New ORM models must be imported in `migrations/env.py` so autogenerate sees
  them.

## 6. Environment configuration

All backend settings are environment variables prefixed `BKN_` (parsed by
`app/core/config.py`). The authoritative list with safe defaults is
`infra/env/.env.dev.example` (dev) and `infra/env/.env.prod.example` (prod).
Never commit a populated `.env`.

## 7. Project structure (top level)

```
backend/    FastAPI app (Clean Architecture; see backend/README.md)
frontend/   Next.js App Router app
infra/      Docker, compose, nginx, env templates, db init, observability
automation/ n8n workflow exports
scripts/    dev-up, deploy, backup
docs/       architecture + this guide
.github/    CI/CD
```

## 8. Pre-commit hooks

```bash
pip install pre-commit && pre-commit install   # repo-wide (Python + frontend)
```

The frontend also ships a Husky + lint-staged hook (`frontend/.husky/pre-commit`)
that runs Prettier and ESLint on staged files. Use whichever your team prefers.
